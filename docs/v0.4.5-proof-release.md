# v0.4.5 Proof Release Gate

`v0.4.5` is an innovation and proof release. Do not publish public speed or BLE claims from this branch unless the evidence below exists.

## Implemented Scope

- Node lifecycle supervisor for relay and LAN-discovery setting changes.
- Transfer-scoped diagnostic bundles from transfer cards.
- Settings readiness cards for route state, node supervisor, benchmark status, and experimental BLE status.
- Clear transfer history and peer-cache controls.
- Experimental BLE status plumbing, off by default. BLE does not move file data.
- Benchmark and physical Android acceptance scripts.

## Required Local Gates

```powershell
pnpm lint
pnpm typecheck
pnpm build
cargo test --manifest-path src-tauri/Cargo.toml --lib
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets -- -D warnings
```

`cargo test --manifest-path src-tauri/Cargo.toml` should also be run in CI/Linux because the Windows integration-test binary can fail before execution on some local setups.

## Required Android Gate

Connect exactly one physical Android phone with USB debugging enabled:

```powershell
.\scripts\android-physical-acceptance.ps1 -CleanInstall
```

Then complete:

- Cold launch, force close, reopen, rotate, background, foreground: no crash.
- Settings -> Copy diagnostics includes Android, Rust, frontend, supervisor, BLE, and transfer context.
- Windows -> Android 10 MB transfer completes.
- Android -> Windows 10 MB transfer completes.
- One 500 MB transfer completes while the phone screen is locked.

## Required Benchmark Gate

Prepare payloads and a CSV template:

```powershell
.\scripts\benchmark-matrix.ps1
```

Record at least five runs for each supported route/scenario before adding speed claims to release notes.

## Release Rule

Do not create or push tag `v0.4.5` until the physical Android gate and benchmark evidence required for any public claims are complete.
